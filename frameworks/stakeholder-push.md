# 协同推动框架

> 在 D 阶段（推动落地 / 卡住了）调用。
> 核心原则：项目成功的关键不是策略有多完美，而是能不能把这件事推得下去。

---

## 协同推动三个层次

```
向上：拿资源和支持
向横：让隔壁团队动起来
向下：让事情真的在城市 / 门店跑起来
```

---

## 向上：怎么拿到资源和支持

**你要回答的四件事：**
1. 为什么要做这件事（价值逻辑）
2. 做这件事的证据是什么（数据支撑）
3. 规划和节奏是什么（让他看到路线图）
4. 预测能带来什么结果（不是承诺，是有数据支撑的预测）

**拿资源的公式：**

```
试点数据证明可行性
+ 横向对比显示优势（比传统模式成本更低、效益更高）
+ 纵向市场分析证明必要性（这件事现在做是窗口期）
+ 竞争态势分析证明紧迫性（竞争对手已经开始了）
= 资源投入的充分理由
```

关键心态转变：**不是在要资源，是在做战略投资建议。**
让领导看到「不投入的风险」比「投入的风险」更大。

---

## 向横：让隔壁团队动起来

你不是他的上级，他没有义务为你服务。关键问题只有一个：
> 你的项目，能不能写进对方的 KPI？

**让对方明确认知到：**
- ✅「我帮你做的这个动作，年底你是能写进绩效里的」
- ✅「这个项目能被你的领导看见」
- ✅「我们一起干成了，功劳是大家的」

**具体推进三步法：**

| 步骤 | 具体行动 | 效果 |
|------|----------|------|
| 主动同步 | 每次有数据、有进展，第一时间同步给他 | 建立正向反馈，让他感受到项目是活的 |
| 提升曝光 | 关键节点抄送他的上级 | 增加他的个人曝光度，让他有参与的动力 |
| 主动服务 | PPT、数据、邮件、月报模板全包 | 让他轻松出彩，降低他配合的成本 |

**核心原则：一切利他。**
不是「请你配合我」，是「你来做这件事，对你有什么好处」。

---

## 向下：让事情在城市 / 门店真的跑起来

大机构里最难的：你是新项目，不是 KPI。你是「创业业务」，要靠自己卖这个项目。

**第一步：把一线当成你要 pitch 的客户**

你要讲清楚：
- 这件事的意义是什么（为什么值得做）
- 对他们有什么好处（不是对公司，是对他个人、对他的团队）
- 前面已经有谁做成了（成功案例比道理更有说服力）

**第二步：打样城市**

先选积极、有意愿、有执行力的城市先试先出结果。
有了第一个成功案例，才有后面「别人都在做，你们落后了」的拉力。

**第三步：飞出去讲故事**

不是发通知，不是群发邮件。是落地去讲：
- 项目背景
- 前面城市怎么做的、出了什么结果
- 这个项目跟他们能有什么回报

**第四步：做氛围 + 内宣**

企业微信、全公司邮件、数据通报——让这个项目「热」起来。
热度就是注意力，有注意力就有人愿意投入。

**落地的本质：让一线愿意干这件事，甚至抢着干。**

---

## 高频卡点诊断

小K看到用户描述的推动难题，先判断卡在哪一层：

| 描述信号 | 判断 | 对应动作 |
|----------|------|----------|
| 领导不批资源 / 不支持 | 向上卡住了 | 补充试点数据 + 竞争态势分析 |
| 隔壁团队不配合 / 不回复 | 向横卡住了 | 找到对他的利益点 + 提升曝光 |
| 城市 / 门店执行不力 | 向下卡住了 | 先打样城市 + 飞出去讲故事 |
| 策略看起来对但推不动 | 利益链条没打通 | 重新梳理每个角色从这件事能得到什么 |

---

## 小K的问题清单（推动落地）

小K先问主人：

1. 现在卡在哪里？（向上 / 向横 / 向下 / 不确定）
2. 对方（领导 / 平级团队 / 城市负责人）目前的态度是什么？
3. 有没有试点数据或成功案例可以用？
4. 对方从这件事里能拿到什么好处——现在说清楚了吗？

用户回答完，小K输出针对这个具体情况的推动方案。

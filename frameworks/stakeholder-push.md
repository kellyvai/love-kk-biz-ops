# 协同推动框架 / Stakeholder Push Framework

> D 入口（推动落地 / 卡住了）调用。
> Used in Entry D (Stuck / Execution Push). Internal reference for diagnosing bottlenecks.

---

## 三层推动 / Three Directions

```
向上 / Up:     拿资源和支持 / Get leadership support & resources
向横 / Across: 让隔壁团队动起来 / Get peer team buy-in
向下 / Down:   让城市/门店跑起来 / Drive frontline execution
```

---

## 向上 / Up: Getting Leadership Support

**四件事必须回答 / Four things you must answer:**
1. 为什么做？（价值逻辑）/ Why? (Value case)
2. 证据是什么？（数据支撑）/ Evidence? (Data that proves it)
3. 规划和节奏？（路线图）/ Plan and timeline? (Roadmap)
4. 预测结果？（有数据支撑的预测）/ Expected outcome? (Data-backed projection)

**心态转变 / Mindset shift**：
不是在要资源，是在做**战略投资建议**。
> You're not asking for resources. You're presenting a strategic investment opportunity.

**公式 / Formula**:
```
试点数据证明可行性 / Pilot data proving viability
+ 横向对比显示优势 / Cross-comparison showing advantage
+ 纵向市场分析证明必要性 / Market analysis showing timing
+ 竞争态势证明紧迫性 / Competition urgency
= 资源投入的充分理由 / Compelling case for investment
```

---

## 向横 / Across: Getting Peer Team Buy-In

你不是他的上级，他没有义务为你服务 / You're not their boss. They have no obligation.

**核心问题 / Core question**:
> 你的项目，能不能写进对方的 KPI？
> Can this project be reflected in their performance metrics?

**推进三步法 / Three-step push**:

| 步骤 / Step | 行动 / Action | 效果 / Effect |
|-------------|--------------|--------------|
| 主动同步 / Sync proactively | 有进展第一时间同步 / Share wins immediately | 建立正向反馈 / Build positive cycle |
| 提升曝光 / Increase visibility | 关键节点抄送他上级 / CC their boss on milestones | 增加他的曝光 / Raises their profile |
| 主动服务 / Serve them | PPT、数据、模板全包 / Do the heavy lifting for them | 降低配合成本 / Remove their friction |

**核心原则 / Core principle**: 一切利他。不是「请你配合我」，是「你来做这件事，对你有什么好处」。
> It's not "please cooperate with me." It's "here's what you get out of this."

---

## 向下 / Down: Driving Frontline Execution

大机构里最难的：你是新项目，不是 KPI / Hardest part in large orgs: you're a project, not a KPI.

**四步法 / Four steps:**

1. **Pitch 一线**：把一线当成客户，讲清楚意义和对ta的好处
   Pitch frontline as customers — what's in it for them?
2. **打样城市**：先选有意愿的城市先出结果
   Pilot with willing teams first, get results
3. **飞出去讲故事**：落地去讲，不是发通知
   Go tell the story in person, don't email it
4. **造势内宣**：企业微信、数据通报，让项目「热」起来
   Build momentum through internal comms

## 卡点诊断 / Bottleneck Diagnosis

| 信号 / Signal | 判断 / Diagnosis | 动作 / Action |
|--------------|-----------------|--------------|
| 领导不批 / Boss won't approve | 向上卡住 / Stuck Up | 补试点数据+竞争分析 / Add pilot data + competitive urgency |
| 平级不回 / Peers won't respond | 向横卡住 / Stuck Across | 找对方的利益点+提升曝光 / Find their win + boost visibility |
| 门店执行不力 / Frontline won't comply | 向下卡住 / Stuck Down | 先打样再讲故事 / Pilot first, then tell stories |
| 说不清卡在哪 / Can't pinpoint | 需要诊断 / Needs diagnosis | 继续问 2-3 个问题 / Ask 2-3 more questions |

---

## 小K的问题清单 / K's Question Flow (D Entry)

1. 现在卡在哪里？（向上/向横/向下/不确定）/ Where are you stuck?
2. 对方现在的态度是什么？/ What's their current stance?
3. 有没有试点数据或成功案例？/ Any pilot data or success stories?
4. 对方从这件事能拿到什么好处——说清楚了吗？/ Have you made clear what they get out of this?

输出：具体推动方案 / Output: specific action plan
